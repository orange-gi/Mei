# Dexter 项目介绍

## 项目概述

Dexter 是一个自主金融研究代理框架，能够像专业分析师一样思考、规划和执行复杂的金融研究任务。该项目采用现代 AI Agent 架构，将大型语言模型的推理能力与实时金融市场数据相结合，为用户提供深度、可靠的金融分析服务。与传统的 Claude Code 等通用代码助手不同，Dexter 专门针对金融领域进行了优化，能够处理股票分析、财务报表解读、投资组合研究等专业任务。

## Why：为什么需要 Dexter

### 金融研究面临的挑战

传统的金融研究方法存在诸多局限性。首先，人工收集和整理财务数据需要耗费大量时间和精力，分析师往往需要在多个数据源之间来回切换。其次，市场信息瞬息万变，手动跟踪公司动态、财务指标变化几乎不可能做到实时响应。再者，复杂的金融建模和估值分析需要深厚的专业背景知识，这对普通投资者或非金融专业人士构成了较高的门槛。最后，研究结果的一致性和可重复性难以保证，不同分析师可能得出截然不同的结论。

### 现有方案的不足

虽然市场上已经出现了一些 AI 助手和金融数据平台，但它们普遍存在以下问题：通用 AI 助手缺乏金融领域的专业知识和工具集成，无法直接访问财务报表、SEC filings 等专业数据源；传统金融数据平台虽然数据丰富，但缺乏智能分析能力，只能提供原始数据而无法生成洞察；现有的 AI Agent 框架大多面向通用编程任务，在金融场景下的应用需要大量定制开发。Dexter 的出现正是为了填补这一空白，提供一个开箱即用的金融研究 AI Agent 解决方案。

### 目标用户群体

Dexter 面向多种用户群体，包括但不限于：投资分析师和研究人员，他们需要快速获取和验证公司财务信息；个人投资者，他们希望获得专业级别的投资研究支持；金融科技开发者，他们需要在应用中集成智能金融分析能力；学术研究者，他们需要进行金融数据分析和模型验证。每个用户群体都可以根据自己的需求定制 Dexter 的功能和输出形式。

## What：Dexter 是什么

### 核心功能定义

Dexter 是一个命令行界面的 AI 代理程序，其核心使命是将复杂的金融问题转化为清晰、可执行的研究计划，并自动完成数据收集、分析和报告生成的全过程。该系统采用迭代式 Agent 架构，支持用户以自然语言提出金融研究请求，然后由 Agent 自动规划任务步骤、调用适当的工具、验证数据准确性，最终输出结构化的研究报告。

### 主要特性

**智能任务规划**：Dexter 能够将用户的复杂查询自动分解为结构化的研究步骤。例如，当用户询问「分析苹果公司和微软公司的投资价值对比」时，Agent 会自动规划出获取两家公司的财务报表、计算关键财务比率、对比估值指标等任务序列。

**自主工具执行**：系统集成了丰富的金融数据工具，包括股票价格查询、财务指标分析、SEC filings 阅读、内部交易追踪等专业功能。Agent 会根据任务需求自动选择合适的工具组合，并在执行过程中进行动态调整。

**自我验证机制**：Dexter 内置了工具调用限制和结果验证逻辑，能够检测重复调用、识别数据异常，并在必要时进行数据交叉验证。这种机制确保了研究结果的可靠性。

**实时市场数据**：通过 Financial Datasets API 等专业数据源，Dexter 能够访问最新的股票价格、财务报表、加密货币价格等市场数据，保证了分析的时效性和准确性。

**多模型支持**：系统支持多种大型语言模型提供商，包括 OpenAI（默认）、Anthropic、Google、xAI（Grok）、OpenRouter、Ollama 等，用户可以根据需求和成本考虑灵活切换。

### 技术差异化

与市面上其他金融分析工具相比，Dexter 具有以下显著差异：首先，它是一个真正的自主 Agent，而非简单的数据查询界面，能够进行多轮推理和迭代分析；其次，它采用了先进的提示词工程技术，将复杂的工具使用规则和专业知识编码到系统提示词中；再者，它提供了完整的技能框架，支持通过 SKILL.md 文件扩展新的分析能力；最后，它内置了评估框架（基于 LangSmith），支持对 Agent 性能进行系统性测试和优化。

## How：Dexter 如何工作

### 快速开始指南

**环境准备**：确保已安装 Bun 运行时环境（v1.0 或更高版本）。如果尚未安装，可以通过以下命令快速安装：

```bash
# macOS/Linux
curl -fsSL https://bun.com/install | bash

# Windows
powershell -c "irm bun.sh/install.ps1|iex"
```

**安装配置**：克隆项目仓库并安装依赖：

```bash
git clone https://github.com/virattt/dexter.git
cd dexter
bun install
```

**配置 API 密钥**：复制环境变量模板并编辑配置文件：

```bash
cp env.example .env
# 编辑 .env 文件，填入以下必需密钥：
# OPENAI_API_KEY=your-openai-api-key
# FINANCIAL_DATASETS_API_KEY=your-financial-datasets-api-key
# 可选密钥：ANTHROPIC_API_KEY, GOOGLE_API_KEY, XAI_API_KEY, EXASEARCH_API_KEY 等
```

**启动程序**：运行以下命令进入交互模式：

```bash
bun start    # 生产模式
bun dev      # 开发模式（支持热重载）
```

### 核心使用方式

启动后，用户可以直接输入金融研究问题开始对话。例如：

```
> 分析特斯拉（TSLA）最近三个季度的财务表现
> 比较苹果公司和微软公司的估值水平
> 帮我做一家公司的 DCF 估值分析
> 查看英伟达最近的 SEC filings 有什么重要信息
```

Agent 会自动解析问题、规划任务、调用工具、收集数据，并最终输出结构化的分析报告。

### 关键配置说明

**模型选择**：Dexter 默认使用 OpenAI 的 GPT-5.2 模型。用户可以通过 `/model` 命令切换到其他模型提供商，支持的模型前缀包括：`claude-`（Anthropic）、`gemini-`（Google）、`grok-`（xAI）、`openrouter:`、`kimi-`（Moonshot）、`deepseek-`、`ollama:`（本地模型）。

**工具配置**：部分工具需要额外的 API 密钥才能启用。Web 搜索功能优先使用 Exa（`EXASEARCH_API_KEY`），如果未配置则回退到 Tavily（`TAVILY_API_KEY`）。本地模型支持需要配置 `OLLAMA_BASE_URL`。

**上下文阈值**：系统设置了 token 使用阈值（默认配置），当上下文超过限制时会自动清理最早的工具调用结果，以防止超出模型的上下文窗口限制。

### 最佳实践建议

**明确具体的问题**：相比宽泛的问题如「告诉我关于苹果公司的一切」，具体的问题如「苹果公司 2024 财年的营收增长情况和毛利率变化」能够获得更精准的回答。

**利用技能系统**：对于需要复杂分析的任务（如 DCF 估值），可以直接请求使用相应的技能，系统会自动识别并调用。

**查看调试日志**：所有工具调用和中间结果都会记录在 `.dexter/scratchpad/` 目录下，可以用于审计和调试。

**参与评估**：可以使用评估套件测试 Agent 在各种金融问题上的表现，帮助改进系统性能。

## 总结

Dexter 代表了金融研究自动化的新一代解决方案。它通过将大型语言模型的强大推理能力与专业金融数据工具相结合，实现了从数据收集到洞察生成的端到端自动化。无论是专业分析师还是普通投资者，都可以通过 Dexter 快速获得高质量的金融研究报告。随着技能框架的不断扩展和评估体系的持续优化，Dexter 有望成为金融领域最受欢迎的 AI 研究助手之一。
