# Personal AI Infrastructure（PAI）项目介绍

## 为什么（Why）

### 解决的问题

PAI（Personal AI Infrastructure）的创建者Daniel Miessler认为，当今世界存在一个核心问题：**地球上只有极少数人的创造潜力被激活**。大多数人并不认为自己有能力做出有价值的贡献，他们认为存在“特殊的人”而自己不属于其中。这种情况使他们在AI时代面临严重的被取代风险。

PAI的存在就是为了解决这个根本问题。项目的使命具有双重性：

1. **激活尽可能多的人**——通过AI增强的自我发现，帮助人们识别、阐明和追求自己的人生目标
2. **让世界上最好的AI人人可及**——确保这种质量的AI基础设施不仅仅为富人或技术精英服务

这就是为什么PAI是一个开源项目，而不是私有的。Daniel Miessler明确表示：“AI应该放大每个人的能力——而不仅仅是前1%的人。”

### 现有方案的不足

当前的AI工具（如ChatGPT、Claude等）存在根本性局限：

- **记忆缺失**：每次对话都是全新的，AI不记得你昨天说了什么
- **无上下文理解**：AI不知道你的目标、偏好或历史
- **工具导向**：大多数代理系统围绕工具构建，用户是事后才考虑的
- **任务而非目标**：现有系统基于任务而非目标，无法理解你真正想要什么

相比之下，PAI将人类放在核心位置，围绕用户的目标、偏好和上下文构建。

## 是什么（What）

### 核心定义

PAI是一个个性化AI平台，旨在放大你的能力。它是为人类设计的，但也可供团队、公司或任何渴望成为更好自己的实体使用。规模不重要——它是一个使用功能全面的代理AI平台来理解、阐明和实现其主人目标的系统。

### 三大核心差异化

PAI与现有代理系统（如Claude Code、Cursor、Windsurf等）有三大核心区别：

1. **目标导向**：PAI的主要焦点是运行它的人类以及他们想在世界上做什么，而不是技术本身。这是系统执行所有任务的方式。

2. **追求最优输出**：系统的外层循环和它所做的一切都是试图产生给定当前情况和所有上下文而言完全正确的输出。

3. **持续学习**：系统不断捕获关于所做的事情、做了什么更改、产生了什么结果的信号，然后是你喜欢或不喜欢的程度。每次交互都会让它更好地帮助你。

### 十六项PAI原则

PAI遵循一套设计原则，涵盖从用户中心性到科学方法的各个方面：

| 原则 | 描述 |
|------|------|
| 用户中心性 | PAI围绕你构建，而非工具。你的目标、偏好和上下文优先 |
| 基础算法 | 科学方法作为通用问题解决循环：观察→思考→计划→构建→执行→验证→学习 |
| 清晰思考优先 | 好的提示来自清晰的思考。在写提示之前先澄清问题 |
| 脚手架大于模型 | 系统架构比使用哪个模型更重要 |
| 确定性基础设施 | AI是概率性的；你的基础设施不应该是。使用模板和模式 |
| 代码优先于提示 | 如果可以用bash脚本解决，就不要用AI |
| 规格/测试/评估优先 | 构建之前先写规格和测试。衡量系统是否有效 |
| UNIX哲学 | 做好一件事。让工具可组合。使用文本接口 |
| ENG/SRE原则 | 像对待生产软件一样对待AI基础设施：版本控制、自动化、监控 |
| CLI即接口 | 命令行界面比GUI更快、更可脚本化、更可靠 |
| 目标→代码→CLI→提示→代理 | 决策层次结构 |
| 技能管理 | 基于上下文智能路由的模块化能力 |
| 记忆系统 | 值得知道的一切都被捕获。历史滋养未来上下文 |
| AI人格 | 不同工作需要不同方法。具有独特声音的专业代理 |
| 科学即元循环 | 假设→实验→衡量→迭代 |
| 失败的许可 | 明确允许说“我不知道”可以防止幻觉 |

### 核心组件（Primitive）

PAI的架构由多个核心组件构成，它们共同创造了与一个理解和了解你的系统合作的体验：

- **TELOS**：10个文件捕获你是谁（MISSION.md、GOALS.md、PROJECTS.md等）
- **用户/系统分离**：你的自定义在USER/，PAI基础设施在SYSTEM/
- **六层自定义**：身份、偏好、工作流、技能、钩子、记忆
- **技能系统**：高度聚焦于一致结果，从CODE→CLI→PROMPT→SKILL
- **记忆系统**：专注于持续学习，三层架构（热/温/冷）
- **钩子系统**：响应生命周期事件，8种事件类型
- **安全系统**：系统和用户级安全策略
- **AI安装**：你的AI助手阅读packs并为你安装一切
- **通知系统**：通过ntfy推送、Discord集成
- **语音系统**：ElevenLabs TTS驱动
- **终端UI**：丰富的标签标题和窗格管理

### 包（Packs）系统

PAI功能以**Packs**的形式分发——即人工智能可安装的自我包含模块。共有23个包：

**基础设施包（5个）**：
- pai-core-install：核心技能、身份、MEMORY系统
- pai-hook-system：事件驱动自动化
- pai-voice-system：ElevenLabs TTS语音通知
- pai-observability-server：实时代理监控仪表板
- pai-statusline：响应式状态行

**技能包（18个）**：
- pai-agents-skill：动态代理组成
- pai-algorithm-skill：ISC管理、努力分类
- pai-annualreports-skill：年度安全报告聚合
- pai-art-skill：视觉内容生成
- pai-brightdata-skill：渐进式URL抓取
- pai-browser-skill：Playwright浏览器自动化
- pai-council-skill：多代理辩论系统
- pai-createcli-skill：生成TypeScript CLI工具
- pai-createskill-skill：创建和验证PAI技能
- pai-firstprinciples-skill：第一性原理分析
- pai-osint-skill：开源情报收集
- pai-privateinvestigator-skill：道德人物查找
- pai-prompting-skill：元提示系统
- pai-recon-skill：安全侦察
- pai-redteam-skill：32个代理对抗分析
- pai-research-skill：多源研究
- pai-system-skill：系统维护和完整性检查
- pai-telos-skill：人生操作系统和深度目标捕获

## 怎么做（How）

### 安装选项

PAI提供四种安装路径：

**选项1：完整发布版安装（推荐）**
这是最快获得工作系统的途径。获得一个完整的预配置的。claude/目录，所有基础设施包已安装。

```bash
# 克隆仓库
git clone https://github.com/danielmiessler/PAI.git
cd PAI/Releases/v2.5

# 备份现有Claude Code配置（如果有）
[ -d ~/.claude ] && mv ~/.claude ~/.claude-backup-$(date +%Y%m%d)

# 复制完整PAI安装
cp -r .claude ~/

# 运行配置向导
cd ~/.claude && bun run INSTALL.ts
```

向导会询问你的名字、AI助手名称和时区，配置环境变量，设置语音偏好（可选），然后验证安装。

**选项2：Bundle + 手动包安装**
适合想要在构建时学习系统或需要自定义设置的用户。

```bash
# 克隆仓库
git clone https://github.com/danielmiessler/PAI.git
cd PAI/Bundles/Official

# 运行交互式向导（创建骨架结构）
bun run install.ts
```

然后必须按顺序安装包：pai-hook-system → pai-core-install → pai-statusline → 其他技能包。

**选项3：单个包安装**
通过给你的AI助手来安装各个包。找到想要的包，给AI助手目录路径，让它安装。

**选项4：浏览和精选**
Packs是自我包含的。可以直接阅读代码，复制特定功能或工作流，方法是适应到自己的系统，或用作参考文档。

### 快速开始

安装后，你可以：

1. **探索你的新系统**——“问我有哪些技能可用”
2. **自定义你的身份**——编辑$PAI_DIR/skills/CORE/USER/文件
3. **设置你的目标**——配置TELOS进行深度目标跟踪
4. **了解工作流**——询问特定功能

### 使用场景

PAI适用于各类用户：

- **小企业主**：处理发票、排程、客户跟进和营销
- **公司**：理解数据、优化运营、做出更好决策
- **经理**：更有效地运行团队——跟踪项目、准备评审、清晰沟通
- **艺术家和创意人员**：寻找本地活动、画廊和展示作品的机会
- **普通人**：改善生活——更好的健身习惯、更强的社交联系、个人财务
- **开发者**：使用AI编码助手想要持久记忆和自定义工作流
- **高级用户**：想要AI知道他们的目标、偏好和上下文
- **团队**：构建具有一致能力的共享AI基础设施
- **实验者**：对AI系统设计和个人AI模式感兴趣

### 技术栈

- **运行时**：Bun
- **语言**：TypeScript、Python、Bash
- **AI平台**：Claude Code（推荐）、Cursor、Windsurf、OpenCode
- **语音**：ElevenLabs TTS
- **平台**：macOS、Linux（完整支持）

## 总结

PAI代表了AI基础设施的范式转变——从将AI视为工具到将其视为了解你、记住你、随着时间推移变得更好的持久助手。通过开源和模块化设计，PAI确保每个人都能获得曾经只有大公司才能拥有的AI能力。
