# GitNexus 项目介绍

## 一、项目概述

GitNexus 是一个为 AI 智能体构建的代码知识图谱工具，其核心理念是「为智能体构建代码上下文」（Building git for agent context）。该项目通过将任意代码库索引为知识图谱，揭示所有依赖关系、调用链、聚类和执行流程，并通过智能工具暴露这些信息，使 AI 智能体能够全面理解代码结构，避免遗漏关键依赖或破坏调用链。

GitNexus 解决了现代 AI 编程助手（如 Cursor、Claude Code、Cline、Roo Code、Windsurf）的一个根本性缺陷：这些工具虽然强大，但并不能真正理解代码库的结构。当 AI 编辑 `UserService.validate()` 时，它不知道有 47 个函数依赖于这个方法的返回类型，这导致了破坏性变更被意外提交。GitNexus 通过预计算的图谱关系，为 AI 智能体提供了完整的代码上下文，使其能够准确评估代码变更的影响范围。

## 二、为什么需要 GitNexus

### 2.1 传统图谱 RAG 的局限性

在 GitNexus 出现之前，开发者尝试使用知识图谱来增强 AI 的代码理解能力，但传统方法存在明显缺陷。传统图谱 RAG 方案向大语言模型提供原始图谱边缘数据，期望模型自行探索足够多的信息。然而，这种方式存在几个严重问题：首先，AI 需要多次查询才能获取完整上下文，这造成了 token 消耗的浪费和响应延迟；其次，AI 可能遗漏关键依赖关系，特别是在复杂的调用链中；第三，小型模型由于缺乏足够的推理能力，往往无法有效利用图谱信息。

### 2.2 AI 智能体的代码理解困境

当前的 AI 编程助手虽然能够生成代码，但在理解代码库结构方面存在根本性障碍。当开发者要求 AI 修改一个函数时，AI 无法准确回答「修改这个函数会影响哪些其他代码」这样的问题。这导致了一系列实际问题：重构时可能破坏隐藏的依赖关系，添加新功能时可能与现有实现冲突，修复 bug 时可能遗漏相关的测试用例。GitNexus 正是为了解决这一核心问题而诞生的。

### 2.3 GitNexus 的创新解决方案

GitNexus 的核心创新在于「预计算关系智能」（Precomputed Relational Intelligence）。与传统方法不同，GitNexus 在索引阶段就完成了聚类、追踪和评分工作，使得工具能够在单次调用中返回完整的上下文信息。这种设计带来了三个关键优势：可靠性方面，AI 不会遗漏上下文，因为所有信息都已在工具响应中预置；Token 效率方面，无需通过多次查询链来理解一个函数；模型民主化方面，较小的模型也能正常工作，因为工具承担了大部分繁重的推理工作。

## 三、GitNexus 是什么

### 3.1 核心功能定位

GitNexus 是一个图谱驱动的代码智能平台，它将代码库转化为可查询的知识图谱，并通过 MCP（Model Context Protocol）协议向外部 AI 智能体提供深度代码理解能力。与 DeepWiki 等代码理解工具不同，GitNexus 不仅仅帮助理解代码，更专注于分析代码——因为知识图谱能够追踪每一种关系，而不仅仅是提供描述性信息。

该工具提供了两种使用方式：CLI + MCP 模式推荐用于日常开发工作，支持 Cursor、Claude Code、Windsurf、OpenCode 等主流编辑器和智能体；Web UI 模式则适用于快速探索和演示场景，用户可以直接在浏览器中可视化管理代码库结构。

### 3.2 支持的编程语言

GitNexus 目前支持九种主流编程语言的代码索引和分析，包括 TypeScript、JavaScript、Python、Java、C、C++、C#、Go 和 Rust。这种广泛的语言支持使得 GitNexus 能够适应大多数开发项目的需求，无论是前端项目、后端服务还是系统级应用。

### 3.3 核心工具能力

GitNexus 通过 MCP 协议暴露了七个核心工具，每个工具都针对特定的代码分析场景进行了优化。`list_repos` 工具用于发现所有已索引的代码库；`query` 工具执行基于进程的分组搜索，结合 BM25 关键词匹配和语义向量搜索，通过倒数排名融合算法返回相关执行流程；`context` 工具提供代码符号的 360 度视图，展示分类的引用关系和参与的进程；`impact` 工具执行爆炸半径分析，计算修改代码符号的影响范围；`detect_changes` 工具分析 Git 差异，映射变更行到受影响的进程；`rename` 工具执行多文件协调重命名，结合图谱和文本搜索；`cypher` 工具支持原始图查询，允许高级用户执行自定义 Cypher 查询。

### 3.4 技术架构概览

GitNexus 的技术栈分为 CLI 和 Web 两个分支。CLI 分支运行在 Node.js 环境中，使用原生 Tree-sitter 进行代码解析，原生 KuzuDB 作为图数据库，HuggingFace transformers.js 生成嵌入向量。Web 分支则完全运行在浏览器中，通过 WebAssembly 技术运行 Tree-sitter WASM 和 KuzuDB WASM，前端采用 React 18、TypeScript、Vite 和 Tailwind v4 构建，使用 Sigma.js 和 Graphology 进行图可视化。

## 四、如何使用 GitNexus

### 4.1 快速开始

对于 CLI + MCP 模式，用户只需在代码库根目录执行以下命令即可完成索引：

```bash
npx gitnexus analyze
```

这条命令会自动完成代码库索引、安装智能体技能、注册 Claude Code 钩子，以及创建 AGENTS.md 和 CLAUDE.md 上下文文件。配置 MCP 只需运行一次 `npx gitnexus setup`，该命令会自动检测编辑器并写入正确的全局 MCP 配置。

### 4.2 编辑器集成

GitNexus 支持多种主流编辑器的 MCP 集成。Claude Code 支持最完整的集成，包括 MCP 工具、智能体技能和 PreToolUse 钩子；Cursor 支持 MCP 和技能功能；Windsurf 支持 MCP 连接；OpenCode 支持 MCP 和技能功能。用户也可以选择手动配置，通过编辑相应的 MCP 配置文件来实现集成。

### 4.3 典型使用场景

在使用 GitNexus 进行代码探索时，典型的流程是先通过 `list_repos` 发现已索引的代码库，然后读取 `gitnexus://repo/{name}/context` 获取代码库概述并检查索引新鲜度，接着使用 `query` 工具搜索相关执行流程，再通过 `context` 工具深入了解特定符号，最后读取 `gitnexus://repo/{name}/process/{name}` 追踪完整执行流程。

在进行影响分析时，首先使用 `impact` 工具查询目标符号的上游依赖，然后读取 `gitnexus://repo/{name}/processes` 检查受影响的执行流程，接着使用 `detect_changes` 工具映射当前 Git 变更到受影响的流程，最后评估风险等级并向用户报告。

### 4.4 知识图谱查询语言

对于高级用户，GitNexus 支持使用 Cypher 查询语言直接查询知识图谱。图谱的节点类型包括 File、Folder、Function、Class、Interface、Method、CodeElement、Community 和 Process，边类型包括 CONTAINS、DEFINES、CALLS、IMPORTS、EXTENDS、IMPLEMENTS、MEMBER_OF 和 STEP_IN_PROCESS。用户可以执行复杂的图查询来获取传统搜索无法提供的信息。

## 五、项目特色与优势

### 5.1 多代码库 MCP 架构

GitNexus 采用全局注册表设计，使一个 MCP 服务器能够同时服务多个已索引的代码库。每次 `gitnexus analyze` 都会将索引存储在代码库内的 `.gitnexus/` 目录中（便携且被 Git 忽略），并在 `~/.gitnexus/registry.json` 中注册一个指针。当 AI 智能体启动时，MCP 服务器读取注册表并可以服务任何已索引的代码库。KuzuDB 连接在首次查询时延迟打开，空闲 5 分钟后关闭，最多支持 5 个并发连接。

### 5.2 智能体技能系统

GitNexus 自动安装四个智能体技能到 `.claude/skills/` 目录，分别是探索技能（使用知识图谱导航不熟悉代码）、调试技能（通过调用链追踪 bug）、影响分析技能（在变更前分析爆炸半径）以及重构技能（使用依赖映射规划安全重构）。这些技能为 AI 智能体提供了结构化的工作流程，确保每次交互都能得到最优结果。

### 5.3 预计算关系智能

GitNexus 的核心创新在于预计算关系智能。与传统图谱 RAG 在运行时动态查询不同，GitNexus 在索引阶段就完成了复杂的关系计算，包括社区检测（使用 Leiden 算法识别功能区域）、进程检测（从入口点追踪执行流程）、置信度评分（评估关系匹配的可靠性）以及混合搜索索引（BM25 加语义加倒数排名融合）。这种设计使得工具能够在单次调用中返回完整、结构化的响应。

### 5.4 安全与隐私保障

GitNexus 高度重视用户数据的安全和隐私保护。CLI 模式下，所有操作都在本地机器上运行，没有任何网络调用，索引存储在 `.gitnexus/` 目录中（已加入 Git 忽略），全局注册表仅存储路径和元数据。Web 模式下，所有操作都在浏览器中运行，代码从不上传到任何服务器，API 密钥仅存储在 localStorage 中。项目采用开源模式，用户可以自行审计代码以验证安全声明。

## 六、总结

GitNexus 代表了 AI 代码助手的下一代发展方向——从单纯的代码生成工具进化为具备深度代码理解能力的智能伙伴。通过将代码库转化为可查询的知识图谱，并提供精心设计的 MCP 工具和技能系统，GitNexus 使 AI 智能体能够真正「理解」代码结构，而不仅仅是生成代码片段。这种范式转变不仅提高了 AI 辅助开发的可靠性，也为小型语言模型提供了接近大型模型的代码理解能力，推动了 AI 开发工具的民主化进程。

无论是日常开发中使用 Cursor 或 Claude Code，还是需要快速探索未知代码库，GitNexus 都能提供有力支持。其开源特性、多语言支持、强大的图谱查询能力以及注重隐私的设计，使其成为现代软件开发不可或缺的智能工具。
