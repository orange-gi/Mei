# Clawdbot 项目介绍

## 项目概述

Clawdbot 是一个个人专属的 AI 助手项目，旨在为用户提供一个可以在自有设备上运行的本地化智能助手解决方案。与传统的云端 AI 服务不同，Clawdbot 强调隐私性、速度和持续可用性，让用户能够完全掌控自己的 AI 助手体验。该项目采用现代化的技术架构，支持跨平台部署和多种消息渠道集成，真正实现了「任何操作系统、任何平台」的愿景。

作为一个开源项目，Clawdbot 展现了开源社区在 AI 助手领域的创新探索。它不仅仅是一个技术产品，更是一种理念的体现——让每个人都能够拥有并控制自己的数字助手，而不必依赖大型科技公司的云端服务。这种设计理念契合了当前用户对数据隐私和自主权的日益增长的关注，为那些希望在日常生活中获得 AI 助手的便利，同时又不希望将个人数据交给第三方的用户提供了一个理想的解决方案。

## 为什么需要 Clawdbot

### 现有解决方案的局限性

在当前的 AI 助手市场中，用户面临着一个根本性的选择困境。一方面是功能强大的云端 AI 服务如 ChatGPT、Claude 等，它们提供了出色的对话能力和持续更新的模型，但同时也意味着用户的所有交互数据都被发送到远程服务器进行分析和存储。这种模式对于关注隐私的用户来说是一个无法回避的问题，尤其是当 AI 助手需要处理敏感信息、进行个人日程管理或者访问用户的其他服务时，数据安全问题变得更加突出。

另一方面，市面上也存在一些本地部署的 AI 解决方案，但它们往往存在使用门槛高、功能单一、用户体验差等问题。许多本地 AI 工具需要用户具备相当的技术背景才能配置和使用，而且通常缺乏与主流通讯平台的无缝集成能力。这意味着用户即使成功部署了本地 AI，也很难将其融入日常的工作流程中，无法真正发挥 AI 助手的潜力。

现有的商业 AI 助手产品还普遍存在另一个问题：平台锁定。大多数 AI 服务只支持特定的平台或渠道，或者需要用户支付高额的订阅费用才能获得完整的功能。这种碎片化的生态使得用户难以在一个统一的界面中管理所有的 AI 交互，也无法根据自己的需求灵活选择和切换不同的服务提供商。

### Clawdbot 的解决思路

Clawdbot 的出现正是为了解决上述这些痛点。它采用了一种独特的设计理念：将 AI 助手的核心能力本地化，同时保持与外部世界的灵活连接。这种设计让用户既能享受本地部署带来的隐私保护和速度优势，又能通过丰富的渠道集成获得便捷的使用体验。项目的目标不是创建一个全新的独立应用，而是让 AI 助手能够融入用户已经熟悉的通讯工具和工作流程中，真正成为用户数字生活的得力助手。

从技术角度来看，Clawdbot 通过 Gateway 架构实现了控制平面与执行平面的分离。Gateway 作为核心控制组件，负责管理会话、处理事件、协调各个模块的工作，而具体的执行任务则可以灵活分配到本地或远程的处理器上。这种架构不仅提供了良好的性能表现，还为未来的扩展和定制提供了充足的空间，让开发者能够根据特定需求对系统进行深度定制。

### 目标用户群体

Clawdbot 主要面向以下几类用户群体。首先是对数据隐私有较高要求的个人用户，他们希望在获得 AI 助手便利的同时，能够完全掌控自己的数据，不让敏感信息离开自己的设备。这类用户通常是技术从业者、企业管理者或者对数字安全有深入了解的人群，他们有足够的技术能力来理解和配置本地部署的解决方案，同时也愿意投入时间学习使用新的工具。

其次是需要高效工作流程的专业人士。Clawdbot 的多渠道集成能力让它能够无缝融入用户的日常工作环境，无论是通过 Slack 与团队协作、通过 Telegram 与客户沟通，还是通过 Discord 参与社区讨论，用户都可以在一个统一的界面中与自己的 AI 助手进行交互。这种跨平台的统一体验大大简化了工作流程，让用户能够在不同的场景之间灵活切换，而不必为每个平台学习不同的 AI 工具。

此外，Clawdbot 也非常适合那些希望探索和实验 AI 技术的开发者和爱好者。作为一个开源项目，Clawdbot 提供了丰富的扩展机制和技能系统，让开发者能够根据自己的需求添加新的功能或者集成新的服务。这种开放的架构不仅满足了高级用户的定制需求，也为整个社区的协作创新提供了基础。

## Clawdbot 是什么

### 核心功能特性

Clawdbot 的核心功能可以概括为四个方面：多渠道通讯、跨平台支持、智能对话和可视化交互。在多渠道通讯方面，项目支持几乎所有主流的即时通讯平台，包括 WhatsApp、Telegram、Slack、Discord、Google Chat、Signal、iMessage、Microsoft Teams、WebChat 等。此外，项目还支持一些扩展渠道如 BlueBubbles、Matrix、Zalo 等，覆盖了全球范围内最常用的通讯工具。这种广泛的渠道支持意味着用户可以在自己已经习惯的平台上与 AI 助手交互，无需切换应用或学习新的界面。

跨平台支持是 Clawdbot 的另一个核心优势。项目能够在多种操作系统上运行，包括 macOS、Linux 和 Windows（通过 WSL2），并且提供了 iOS 和 Android 移动端的客户端支持。这种全面的平台覆盖让用户无论使用什么设备，都能够访问自己的 AI 助手。更重要的是，项目采用了统一的设计理念和交互模式，让用户在不同设备上都能获得一致的使用体验。项目的推荐安装方式是通过 CLI 向导（`clawdbot onboard`）进行配置，这个向导会引导用户完成 Gateway、workspace、channels 和 skills 的设置，整个过程简单直观。

智能对话功能是 Clawdbot 作为 AI 助手的核心能力。项目支持多种 AI 模型，包括 Anthropic 的 Claude 系列和 OpenAI 的 GPT 系列。项目特别推荐使用 Anthropic Pro/Max 订阅配合 Opus 4.5 模型，以获得最佳的长上下文处理能力和更强的提示注入攻击防护能力。用户可以根据自己的需求选择不同的模型，也可以配置模型故障转移机制，在主模型不可用时自动切换到备用模型，确保服务的连续性。

可视化交互是 Clawdbot 区别于传统文本对话型 AI 助手的独特特性。项目支持 Canvas 渲染功能，能够生成实时的可视化内容供用户查看和交互。这种能力在需要图形化展示信息、数据可视化或者创意表达的场特别有用，大大扩展了 AI 助手的应用场景。语音功能也是项目的重要组成部分，Clawdbot 能够在 macOS、iOS 和 Android 平台上支持语音输入和输出，为用户提供更加自然的交互方式。

### 技术架构概述

Clawdbot 采用现代化的技术栈构建，核心代码以 TypeScript 为主，占项目总代码量的约 79.8%。这种选择不仅保证了代码的类型安全性，还使得项目具有良好可维护性和可扩展性。对于 iOS 移动端，项目使用 Swift 进行开发（占 13.4%），充分利用了 Apple 平台的原生能力和性能优势。Android 端采用 Kotlin（占 1.9%），而前端界面则使用 JavaScript（占 3.2%）实现。这种多语言、多平台的技术组合展示了项目在跨平台开发方面的深厚积累。

项目的包管理使用 pnpm，这是一个高性能的 Node.js 包管理器，能够有效节省磁盘空间并加速依赖安装。项目管理采用了 pnpm workspace 模式，支持单一仓库中的多个子项目管理，便于代码共享和版本协调。构建系统支持多种输出目标，能够生成适用于不同平台和部署方式的产物。

从架构设计的角度来看，Clawdbot 采用了清晰的模块化结构。项目的主要目录包括：`src/` 存放核心源代码，`apps/` 包含应用程序模块，`extensions/` 存放扩展功能模块，`skills/` 是技能插件目录，`ui/` 包含用户界面相关代码。这种组织方式使得各个功能模块之间保持低耦合，便于独立开发、测试和维护。Gateway 组件作为系统的控制平面，负责处理会话管理、通道连接、工具协调和事件分发等核心功能，是整个系统的枢纽。

### 与竞品的差异化

相比市场上其他的 AI 助手解决方案，Clawdbot 具有几个显著的差异化特点。首先是真正的本地化部署能力。与许多标榜「本地」但实际上仍然依赖云端服务的解决方案不同，Clawdbot 的核心运行在用户的自有设备上，用户对自己的数据和模型拥有完全的控制权。这种设计不仅提供了更好的隐私保护，还能够在网络不可用时继续提供服务，保证了 AI 助手的高可用性。

其次是广泛的多渠道集成。很少有 AI 助手能够同时支持如此多的通讯平台，而且不是简单地提供 API 集成，而是深度适配每个平台的特点和用户交互习惯。这种广泛的渠道支持让用户能够真正将 AI 助手融入日常工作流，而不是需要专门打开另一个应用来使用 AI 功能。项目还支持 BlueBubbles、Matrix 等相对小众但具有特定用户群体的平台，展现了对用户多样化需求的尊重。

第三是开放的扩展生态。Clawdbot 提供了技能（Skills）系统，允许开发者和用户创建自定义的功能模块。这些技能可以是简单的脚本，也可以是复杂的集成服务，既有本地运行的技能，也有通过云端提供的技能。这种开放的生态设计让 Clawdbot 能够不断扩展其能力边界，而核心系统的维护负担并不会线性增长。社区贡献的技能可以被其他用户复用，形成了良性的协作循环。

## 如何使用 Clawdbot

### 环境准备与安装

Clawdbot 的运行环境要求 Node.js 版本大于等于 22。这是一个相对较高的版本要求，确保了项目能够使用最新的 JavaScript 语言特性和 Node.js 性能优化。安装过程非常简单，推荐使用 npm 或 pnpm 进行全局安装：

```bash
npm install -g clawdbot@latest
# 或者使用 pnpm
pnpm add -g clawdbot@latest
```

安装完成后，项目推荐使用引导式安装命令 `clawdbot onboard` 来完成初始配置。这个命令行向导会逐步引导用户完成 Gateway 配置、工作区设置、渠道连接和技能选择等关键步骤。向导的设计非常人性化，即使是缺乏技术背景的用户也能够顺利完成设置。对于高级用户，向导也提供了跳过某些步骤或手动配置高级选项的选项。

在安装过程中，向导会询问是否安装 Gateway 守护进程（daemon）。这个守护进程是一个用户级的系统服务（macOS 使用 launchd，Linux 使用 systemd），能够让 Clawdbot 在后台持续运行，即使断开 SSH 会话或关闭终端也不会影响服务的运行。强烈建议在生产使用时安装这个守护进程，以确保 AI 助手能够随时响应用户的请求。

### 快速启动指南

安装完成后，启动 Clawdbot 的最简方式是运行以下命令：

```bash
clawdbot gateway --port 18789 --verbose
```

这条命令会启动 Gateway 服务，默认监听 18789 端口。`--verbose` 参数会让服务输出详细的日志信息，便于调试和问题排查。启动成功后，用户可以看到服务正在运行的提示信息，此时 Clawdbot 已经准备好接收用户的指令了。

要向 AI 助手发送消息，可以使用 `message` 命令：

```bash
clawdbot message send --to +1234567890 --message "Hello from Clawdbot"
```

这个命令会通过配置的渠道向指定的接收者发送消息。`--to` 参数指定接收方的标识，格式因渠道而异（例如电话号码用于 WhatsApp/Signal，用户名用于 Telegram/Discord）。如果需要与 AI 助手进行对话，可以使用 `agent` 命令：

```bash
clawdbot agent --message "Ship checklist" --thinking high
```

`--thinking high` 参数会让助手在回答时显示更多的思考过程，这对于理解 AI 的推理逻辑或调试问题很有帮助。用户也可以省略这个参数，让助手直接给出简洁的回答。

### 配置管理详解

Clawdbot 的配置主要通过命令行参数和环境变量进行管理，同时也支持通过配置文件进行持久化设置。Gateway 的核心配置包括端口号、日志级别、工作目录等常用参数。用户可以在首次运行时通过向导进行配置，也可以直接编辑配置文件进行高级定制。

模型配置是 Clawdbot 中最重要的设置之一。用户需要配置至少一个 AI 模型提供商的认证信息才能使用对话功能。项目支持两种认证方式：OAuth 授权和 API 密钥。OAuth 方式适用于 Anthropic 和 OpenAI 的官方订阅服务，能够自动处理令牌的刷新和过期问题；API 密钥方式则更加灵活，适用于自定义的模型服务或第三方提供商。模型配置还支持故障转移设置，用户可以配置多个模型作为备用，在主模型不可用时自动切换。

渠道配置决定了 AI 助手能够通过哪些平台与用户交互。每个渠道都有其特定的认证要求和使用限制。例如，Discord 需要创建机器人并获取令牌，WhatsApp 需要通过 QR 码进行配对，Telegram 需要联系 BotFather 获取 API 密钥。向导会为每个渠道提供详细的配置指导，用户也可以参考项目的在线文档获取更详细的信息。配置完成后，渠道会在后台持续运行，自动处理消息的接收和发送。

### 技能系统使用

技能（Skills）是 Clawdbot 扩展功能的核心机制。用户可以通过技能系统添加新的功能模块，让 AI 助手能够执行特定的任务或与特定的服务进行集成。项目预置了一些常用技能，同时也支持用户创建自定义技能。技能的来源分为本地技能和云端技能两类：本地技能直接运行在用户的设备上，具有更好的隐私性和响应速度；云端技能则运行在远程服务器上，可以提供更强大的计算能力或访问特定的网络服务。

使用技能时，用户只需要在对话中自然地提出需求，AI 助手会根据对话内容自动判断是否需要调用某个技能，以及应该使用哪个技能。例如，如果用户说「帮我查一下今天的天气」，助手会自动调用天气相关的技能，获取并返回天气信息。这种设计让技能的调用变得透明和自然，用户无需了解底层的技术细节。

对于开发者来说，创建自定义技能的过程相对简单。技能本质上是一个定义了特定接口的模块，需要实现如 `description`、`parameters`、`handler` 等标准属性。描述属性用于说明技能的功能和使用方式；参数属性定义了技能接受的输入格式；处理函数则是技能的实际执行逻辑。项目提供了详细的技能开发文档和示例代码，帮助开发者快速上手。

### 最佳实践建议

为了获得最佳的使用体验，这里提供一些使用 Clawdbot 的最佳实践建议。首先是关于模型选择的建议：虽然 Clawdbot 支持多种 AI 模型，但项目特别推荐使用 Anthropic 的 Claude Pro 或 Max 订阅配合 Opus 4.5 模型。这一组合在长上下文处理、指令遵循和安全性方面都有出色的表现，能够为用户提供最接近理想状态的 AI 助手体验。当然，如果用户已经有其他模型的订阅或偏好，也可以自由选择，项目不会强制用户使用特定的模型。

其次是关于渠道管理的建议。建议用户根据自己的实际使用场景选择和配置渠道，而不必一次性启用所有支持的渠道。过多的渠道可能会增加维护的复杂性，也可能导致消息来源过于分散，影响使用效率。通常，选择一到两个最常用的渠道进行配置即可满足大多数需求。如果确实需要跨多个渠道使用 AI 助手，建议在每个渠道上设置清晰的标识，避免在不同渠道上混淆对话上下文。

第三是关于系统维护的建议。Clawdbot 作为长期运行的服务，需要定期进行维护和更新。项目提供了 `clawdbot doctor` 命令用于诊断常见的配置问题和建议修复措施。定期运行这个命令可以帮助用户发现和解决潜在的问题，保持系统的健康状态。更新 Clawdbot 本身也非常简单，只需要通过包管理器安装最新版本即可，项目文档中提供了详细的升级指南。需要注意的是，升级前建议备份重要的配置和数据，以防升级过程中出现意外情况。
