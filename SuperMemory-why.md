# Supermemory 项目介绍

## 为什么（Why）

### 问题背景与动机

在当今信息爆炸的数字时代，用户每天都会接触到海量的信息内容，包括文章、推文、文档、视频、聊天记录等等。然而，这些宝贵的信息往往散落在不同的平台和应用中——浏览器书签、社交媒体、笔记应用、邮件附件、云端文档——形成一个又一个信息孤岛。当用户真正需要检索和利用这些历史信息时，却发现传统的信息管理工具（如浏览器书签、系统文件管理器）根本无法提供智能的检索和关联能力。信息的保存与信息的再利用之间存在着巨大的鸿沟，这不仅造成了认知负担的加重，也导致了大量有价值知识的流失和遗忘。

更为关键的是，随着人工智能助手（如Claude、ChatGPT、Cursor等）在我们日常工作流程中的普及，这些AI工具面临着严重的「记忆缺失」问题。每一次新的对话都是从头开始，AI无法记忆之前的上下文、个人偏好、工作风格或已经处理过的信息。这种「短期记忆」的限制严重影响了AI辅助工作的效率和质量。用户不得不反复提供背景信息、解释项目上下文、重新说明自己的偏好，导致了大量的时间和精力的浪费。

### 现有方案的不足

传统的信息管理工具和知识库系统存在多方面的局限性。首先，现有的书签工具（如浏览器书签、Pinterest收藏）仅能保存链接和简单描述，缺乏对内容语义的理解和智能检索能力，用户只能依赖模糊的关键词记忆来查找信息，往往事倍功半。其次，传统的笔记应用（如Evernote、Notion）虽然提供了丰富的内容组织功能，但其搜索能力仍停留在关键词匹配层面，无法理解用户的查询意图和内容的深层语义关系。第三，单纯的向量数据库解决方案（如Pinecone、Weaviate）虽然提供了语义搜索能力，但缺乏完整的数据采集、内容处理、用户交互层，对于普通用户而言技术门槛过高。

另一方面，针对AI助手的记忆解决方案也各有缺陷。简单地在提示词中附加历史信息会导致上下文窗口溢出，昂贵且低效；使用外部向量存储检索相关内容则需要复杂的系统集成，开发成本高昂；而现有的开源记忆项目往往性能低下、扩展性差、用户体验不佳，难以真正作为生产级系统部署。

### 目标用户群体

Supermemory的核心目标用户群体包括以下几类：

**知识工作者与研究者**：包括产品经理、研究员、咨询顾问、学术研究者等需要大量收集、整理和检索信息的专业人士。他们每天都会接触大量的研究报告、行业动态、学术论文，需要一个能够快速保存、智能索引、随时检索的「第二大脑」来辅助工作。

**开发者与程序员**：特别是使用AI辅助编程工具（如Cursor、Windsurf、Claude Code）的开发者群体。他们需要在代码库之外维护项目上下文、技术选型决策、设计文档等信息的长期记忆，使AI能够真正理解项目背景并提供更精准的建议。

**内容创作者与营销人员**：包括博主、自媒体运营者、营销策划人员等需要持续追踪行业动态、收集素材、管理知识资产的内容工作者。Supermemory能够帮助他们快速保存灵感素材、整理竞品分析、构建内容资产库。

**AI爱好者与高级用户**：对AI工具有深入了解，希望构建个性化AI工作流的极客用户。他们愿意尝试新工具，并希望通过记忆系统来增强AI的个人化能力和工作效率。

## 是什么（What）

### 项目概述

Supermemory是一款专为AI时代设计的极速、可扩展的记忆引擎和应用程序。它既是一个面向普通用户的Web应用和浏览器扩展，允许用户从任意来源（网页、PDF、文本、Notion、Google Drive等）保存内容并与之对话；也是一个面向开发者的Memory API，可以通过MCP协议无缝集成到任何AI工具（Claude、Cursor、ChatGPT等）中，为AI提供长期记忆能力。

作为一个完整的「AI记忆层」解决方案，Supermemory构建在高性能基础设施之上：使用PostgreSQL作为核心数据存储，结合Cloudflare Durable Objects构建的自定义向量引擎，能够支持每位用户高达5000万token的存储规模，端到端延迟控制在400毫秒以内。这种架构设计使得Supermemory既具备企业级的可靠性和扩展性，又保持了开源项目的灵活性和可自部署性。

### 核心功能特性

**多源内容采集**：Supermemory支持从多种来源快速添加记忆内容。用户可以直接粘贴文本、上传PDF文件、导入网页链接、连接Notion工作区、Google Drive或OneDrive云端存储，实现一站式的信息收集。所有采集的内容都会经过自动化的解析、清洗和向量化处理，保留原始内容的完整语义信息。

**语义智能搜索**：基于向量相似度搜索技术，Supermemory能够理解用户的查询意图，从海量记忆中找到语义相关的内容片段。无论是精确关键词搜索、自然语言问题查询还是概念联想检索，Supermemory都能返回高度相关的记忆片段，并标注相关度得分。

**对话式记忆交互**：用户可以像与ChatGPT对话一样与自己的记忆库进行交互。系统会检索相关的历史记忆片段，将它们作为上下文提供给大语言模型，生成基于用户个人知识库的智能回答。这种「与自己的记忆对话」的体验，让知识的再利用变得自然流畅。

**MCP协议集成**：通过Model Context Protocol（MCP），Supermemory可以无缝连接到Claude Desktop、Cursor、Windsurf、VS Code、ChatGPT等主流AI工具。一旦连接成功，AI工具就能实时检索用户的记忆库，将个人上下文融入每一次交互中，无需手动复制粘贴或管理提示词模板。

**浏览器扩展支持**：Supermemory提供Chrome/Edge浏览器扩展，允许用户在任何网页上快速保存内容。扩展还集成了与ChatGPT和Claude对话的快捷入口，以及Twitter/X推文导入功能，让信息捕捉在浏览过程中无缝发生。

**Raycast快捷入口**：对于使用Raycast启动器的macOS用户，Supermemory提供了专用的Raycast扩展，支持通过快捷命令添加记忆、搜索内容、管理记忆库，实现键盘驱动的高效工作流。

### 技术栈概述

Supermemory采用现代化的全栈技术架构，构建在以下核心技术之上：

**运行时与构建工具**：使用Bun作为JavaScript运行时和包管理器，提供比Node.js更快的启动速度和依赖安装性能。项目采用Turborepo进行单体仓库管理，实现高效的跨包构建和发布流程。

**后端与API框架**：核心后端服务基于Hono框架构建，这是一个轻量级、高性能的Web框架，兼容Cloudflare Workers、Node.js、Deno等多种运行时。API层采用OpenAPI规范文档化，结合zod进行数据验证，确保接口的类型安全和文档完整性。

**数据库与ORM**：使用PostgreSQL作为主数据库存储用户数据、会话信息、元数据等结构化内容。通过Drizzle ORM进行数据库操作，提供类型安全的SQL构建体验和高效的查询性能。Cloudflare KV用于缓存和键值存储。

**AI与向量引擎**：项目深度集成多家主流AI服务商的SDK，包括OpenAI、Anthropic、Google Generative AI等，支持灵活的模型选择和切换。自研的向量引擎基于Cloudflare Durable Objects构建，在边缘节点提供低延迟的向量相似度检索能力。

**认证与安全**：采用better-auth进行用户认证管理，支持多种登录方式和OAuth集成。Sentry用于错误监控和性能追踪，PostHog用于产品分析，全方位保障系统可观测性。

**前端与UI**：Web应用基于Remix框架构建，提供服务端渲染和优秀的首屏加载性能。UI组件采用TailwindCSS进行样式设计，使用Vanilla Extract实现CSS-in-JS的组件级样式管理。多个应用共享UI组件包（packages/ui），确保视觉一致性和代码复用。

## 怎么做（How）

### 快速开始指南

对于普通用户，使用Supermemory的最简方式是访问官方Web应用：

1. 打开浏览器访问 [app.supermemory.ai](https://app.supermemory.ai)
2. 使用GitHub或Google账号登录（更多登录方式即将支持）
3. 开始添加你的第一条记忆：选择笔记、链接或文件格式
4. 添加完成后，点击「Open Chat」与记忆库进行对话

对于希望集成Supermemory到AI工具中的用户，按照以下步骤配置MCP连接：

1. 访问应用中的「Connect to your AI」页面
2. 选择你想要连接的AI工具（Claude Desktop、Cursor等）
3. 按照页面指引获取MCP服务器URL
4. 在对应AI工具的MCP设置中添加服务器配置
5. 连接成功后，即可开始使用「共享记忆」功能

### 浏览器扩展使用

Chrome/Edge浏览器扩展提供了更便捷的记忆保存方式：

1. 从 [Chrome网上应用店](https://chromewebstore.google.com/detail/supermemory/afpgkkipfdpeaflnpoaffkcankadgjfc) 安装扩展
2. 点击扩展图标打开弹窗，选择保存当前页面或选定内容
3. 右键菜单中也提供「Save to Supermemory」快捷选项
4. 扩展会自动抓取页面内容并进行向量化处理

### Raycast扩展使用

macOS用户可以通过Raycast快速访问Supermemory：

1. 在Raycast商店中搜索「Supermemory」并安装扩展
2. 使用快捷键调出Raycast，输入命令进行操作
3. 「Add Memory」命令：快速添加文本或链接到记忆库
4. 「Search Memories」命令：检索个人记忆库

### 核心使用方式

**添加记忆内容**：

Supermemory支持三种主要的记忆添加方式。笔记模式允许用户直接输入文本内容，适用于捕捉灵感、记录想法；链接模式通过URL抓取网页内容，系统会自动提取正文、标题、作者等元信息；文件模式支持PDF、Markdown等格式的文件上传，系统会解析文档结构并提取文本内容。此外，用户还可以连接外部服务（Notion、Google Drive等）进行批量导入。

**管理与组织记忆**：

所有保存的记忆都会显示在主界面列表中，支持按时间、来源、标签等维度筛选。用户可以为记忆添加标签、编写摘要、设置重要性等级，方便后续检索和整理。记忆详情页展示原始内容、向量信息、相关记忆推荐等完整信息，支持编辑和删除操作。

**对话式检索**：

点击「Open Chat」进入对话界面，用户可以用自然语言提问或描述需求。系统会执行以下流程：首先基于用户输入计算向量查询，在记忆库中找到语义相关的片段；然后将相关记忆与用户问题组合，构造完整的提示词；最后调用大语言模型生成回答。结果中会标注引用的记忆来源，方便用户回溯原始内容。

**MCP工具调用**：

连接MCP后，AI工具会自动获得以下能力：搜索记忆（search_memories）用于检索相关内容；添加记忆（add_memory）用于保存新信息；获取配置文件（getProfile）用于读取用户偏好设置。AI会在需要时自动调用这些工具，将用户的个人记忆融入对话上下文中。

### 关键配置说明

**环境变量配置**：

对于自部署用户，需要配置以下关键环境变量：

```bash
# 数据库连接
DATABASE_URL="postgresql://user:password@host:5432/supermemory"

# AI服务API密钥（根据使用的模型选择配置）
OPENAI_API_KEY="sk-..."
ANTHROPIC_API_KEY="sk-ant-..."
GOOGLE_API_KEY="..."

# Cloudflare相关
CLOUDFLARE_ACCOUNT_ID="..."
CLOUDFLARE_API_TOKEN="..."
CLOUDFLARE_KV_NAMESPACE="..."

# 认证配置
AUTH_SECRET="your-secret-key"
```

**自定义模型配置**：

Supermemory支持切换不同的AI模型以平衡性能和成本。默认配置使用Anthropic Claude模型进行对话生成，用户可以在设置页面修改为OpenAI GPT-4、Google Gemini或其他兼容OpenAI API的模型。向量嵌入模型同样支持自定义，推荐使用text-embedding-3-small以获得良好的性价比。

### 最佳实践建议

**记忆组织策略**：

有效的记忆管理需要建立清晰的组织结构。建议为不同项目、领域或时间周期的记忆设置统一的标签体系；定期回顾和整理已保存的内容，删除过时或不再相关的信息；对于重要资料，添加详细的摘要和关键词标签以提升检索准确率。

**MCP集成优化**：

为了获得最佳的AI集成体验，建议在AI工具的对话初期明确说明「请使用我的记忆库」；在提出专业领域问题时，提供适当的问题背景描述以提升检索精度；定期检查MCP工具调用日志，确保记忆检索正常工作。

**性能与成本控制**：

对于高频使用的用户，建议关注token消耗统计，合理设置向量检索的相似度阈值以平衡召回率和精确率；对于不需要长期保留的临时信息，使用「临时笔记」功能避免占用长期存储空间。
